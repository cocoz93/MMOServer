④ 자료구조 · 왜 트리가 아닌가
균일 그리드 vs 트리 (quadtree / k-d)
균일 분포 전제 → O(1) 인덱스·캐시 연속이 트리 동적 분할보다 단순·빠름
균일 그리드 · 채택 ✓
flat vector — 연속 메모리 (캐시 라인 친화)
주변 조회 = y*countX + x
삽입/삭제O(1) push_back / swap-pop
주변 조회O(1) 인덱스 산술
캐시지역성 높음 (평탄화 vector)
균일 분포에 최적
quadtree / k-d tree
노드마다 포인터 추적 — 흩어진 배치
삽입/삭제O(log N) + 분할·병합
주변 조회트리 순회
캐시지역성 낮음 (포인터 추적)
동적 밀집(보스 레이드)에 강함
균일 분포·고정 셀 전제에선 그리드의 단순함이 곧 성능 · 국소 과밀 시 한 섹터 비대는 트레이드오프(약점)
